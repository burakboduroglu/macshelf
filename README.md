## Overview

MacShelf is a lightweight macOS clipboard history app built with SwiftUI,
SwiftData, AppKit, and a global hotkey. It lives quietly in the menu bar,
captures copied text and images, and lets you paste previous items back into the
frontmost app without breaking your flow.

```text
Menu bar -> MacShelf

+--------------------------------+
| Search clipboard...            |
|--------------------------------|
| 1  hello@example.com     now   |
| 2  Image 1200 x 800 - A1B2C3  |
| 3  curl https://...   5 min ago|
+--------------------------------+
| 18 items                    ...|
+--------------------------------+
```

## Features

- Text and image clipboard history
- Native menu bar popover powered by `NSStatusItem` and `NSPopover`
- Global hotkey, default `Cmd+Shift+V`
- Search, arrow-key navigation, Return to paste, Escape to close
- `Cmd+1` through `Cmd+9` quick paste shortcuts
- Press-and-hold Space preview for the hovered item
- Image thumbnails and large Quick Look-style previews
- Pin, unpin, delete, clear history, and copy without auto-paste
- Rounded relative timestamps, with no distracting seconds
- Configurable history limit, with pinned items preserved
- Strict duplicate suppression for text and images
- Self-write suppression, so selected history items are not re-captured
- Privacy skips for password-manager and transient pasteboard items

## Requirements

- macOS 14 or newer
- Xcode with Swift 5.10 support
- Accessibility permission for automatic paste into the focused app

MacShelf still works without Accessibility permission, but selecting an item only
copies it to the system clipboard. You can then press `Cmd+V` manually.

## Build

This repository vendors
[`KeyboardShortcuts`](https://github.com/sindresorhus/KeyboardShortcuts) under
`Vendor/KeyboardShortcuts`, so the app can be built without downloading SwiftPM
dependencies.

### Build a launchable app bundle

```bash
scripts/build.sh
```

The script compiles the Swift package, creates `build/MacShelf.app`, copies the
required `Info.plist` and resource bundles, and ad-hoc signs the app.

To relaunch after building:

```bash
scripts/build.sh --run
```

### Create a DMG

```bash
scripts/package-dmg.sh
```

The DMG is written to `dist/MacShelf-<version>.dmg`. It contains `MacShelf.app` and an `/Applications` shortcut so users can drag the app into Applications.

### Open in Xcode

```bash
open Package.swift
```

Select the `MacShelf` scheme, choose "My Mac", then run.

### About `swift run`

`swift run MacShelf` launches a bare executable, not a proper `.app` bundle.
Some macOS services expect a main bundle identifier, so for daily use prefer
`scripts/build.sh` or Xcode.

## Usage

1. Launch `build/MacShelf.app`.
2. Click the menu bar icon or press `Cmd+Shift+V`.
3. Move the mouse over an item to highlight it.
4. Hold Space to preview the hovered item.
5. Press Return, click an item, or use `Cmd+1` to `Cmd+9` to paste.

Settings are available from the footer menu. From there you can change the
history limit, record a different global hotkey, check Accessibility status, and
open System Settings.

## Privacy

MacShelf skips pasteboard entries marked with:

- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.TransientType`
- `org.nspasteboard.AutoGeneratedType`

It also ignores a small built-in list of known password-manager bundle IDs.

When MacShelf writes one of its own history items back to the system clipboard,
it marks that write as auto-generated. The clipboard monitor ignores that marker
so the same item is not captured again.

## How It Works

macOS does not provide a notification for every clipboard change, so
`ClipboardMonitor` polls `NSPasteboard.general.changeCount` every 500 ms.

For text items, MacShelf stores the exact string in SwiftData. For image items,
it reads common pasteboard image types, file URLs pointing to images, and AppKit
image payloads, then stores PNG data plus dimensions and a SHA-256 hash.

Duplicate handling is strict:

- Same text: ignored
- Same image hash: ignored
- MacShelf's own pasteboard writes: ignored

Pasting writes the selected item back to `NSPasteboard.general`. If Accessibility
permission is granted, MacShelf posts a synthetic `Cmd+V` with `CGEvent`.

## Project Layout

```text
Sources/MacShelf
  App/
    MacShelfApp.swift        SwiftUI app entry and Settings scene
    AppDelegate.swift        Status item, popover, hotkey, monitor lifecycle
  Models/
    ClipboardItem.swift      SwiftData model for text and image entries
  Services/
    ClipboardMonitor.swift   Pasteboard polling, decoding, dedupe, pruning
    PasteService.swift       Pasteboard write and optional Cmd+V event
    PermissionsService.swift Accessibility status and settings shortcut
    HotkeyManager.swift      KeyboardShortcuts registration
  Views/
    MenuView.swift           Search, history list, key capture, preview state
    ItemRow.swift            A single clipboard history row
    DetailsCard.swift        Space preview card
    SettingsView.swift       General, Shortcuts, Privacy tabs
  Resources/
    Info.plist
    MacShelf.entitlements
    Assets.xcassets
```

## Notes

- MacShelf is a menu bar app and does not show a Dock icon.
- The generated `build/MacShelf.app` is ad-hoc signed by the local build script.
- The generated `dist/MacShelf-<version>.dmg` is suitable for GitHub Releases.
- Distribution builds should be signed and notarized with your Apple Developer ID.
- Console messages from macOS services such as ViewBridge or Control Center can
  appear during development and are usually harmless unless the app is visibly
  failing.
